# 동료 평가 기반 커리어 프로필 서비스 - 프리토타이핑 실행 방안

## 서비스 개요

**한 줄 요약:** 재직 중 팀 전체 익명 동료 평가를 축적하여, 이직 시 커리어 프로필로 활용할 수 있는 B2B SaaS 플랫폼

**핵심 플로우:**
1. 회사가 서비스 도입 (B2B)
2. 팀원 전체가 서로 익명 평가 (구조화된 항목)
3. 재직 기간 동안 평가 데이터 축적
4. 퇴사/이직 시 → 커리어 프로필로 활용
5. 채용사가 지원자 동의 하에 평가 열람

---

## 1단계: 프리토타이핑 전략

- **방식:** MVP 직접 개발
- **MVP 범위:** 평가 수집 + 외부 공유용 프로필 생성
- **비즈니스 모델:** B2B (회사 단위 도입)
- **평가 방식:** 팀/회사 단위 전체 평가 (선택 편향 제거)

---

## 2단계: 핵심 타겟

### 1차 타겟 (회사)
- 스타트업 (10~50명 규모)
- 이직률 높은 업종 (IT, 마케팅, 콘텐츠 등)
- 동료 평가 문화에 관심 있는 회사

### 의사결정권자
- CEO / COO (소규모)
- HR 담당자 / People팀

### 타겟 직원 (사용자)
- 경력 2년 이상 (신입 제외)
- 이직 경험 또는 이직 고려 중인 직장인

---

## 3단계: 핵심 메시지 및 가치

### 핵심 메시지
> "동료 평가가 만드는 건강한 조직 문화"

### 서브 메시지
- 평가가 남으면, 행동이 달라집니다
- 업무 태도 향상 → 성과↑
- 동료 관계 신경 → 협업↑, 갈등↓

### 핵심 차별점 (vs 기존 서비스)

| 구분 | 기존 서비스 (스펙터 등) | 우리 서비스 |
|------|------------------------|------------|
| 평가 시점 | 채용 시 (사후, 1회성) | 재직 중 (사전, 상시 축적) |
| 평가자 선정 | 지원자가 직접 부탁 (편향) | 팀 전체 자동 참여 (객관) |
| 신뢰도 | 낮음 (친한 사람만) | 높음 (전원 익명) |
| 부담감 | 매번 부탁, 이직 노출 위험 | 자연스럽게 축적 |
| 1차 가치 | 채용 검증 | 조직 문화 개선 |
| 2차 가치 | - | 이직 시 포트폴리오 |

## 4단계: 콘텐츠 제작안

### A. 랜딩페이지

```
[히어로 섹션]
- 헤드라인: "동료 평가가 만드는 건강한 조직 문화"
- 서브: 평가가 남으면, 행동이 달라집니다
- CTA: "도입 문의하기"

[문제 제기]
- "피드백 없이 성장할 수 있을까요?"
- "퇴사하면 사라지는 동료의 신뢰"

[기존 서비스의 한계]
- 지원자가 직접 추천인 부탁 → 편향된 평가
- 1회성 레퍼런스 체크 → 데이터 부족

[우리의 솔루션]
- 목업 화면 1~2개
- 핵심 기능: 팀 전체 익명 평가 → 데이터 축적 → 프로필 생성

[도입 효과]
- 협업↑ 갈등↓ 성과↑

[CTA]
- 도입 문의 폼 (회사명, 담당자, 이메일)
```

### B. 목업 화면
1. **평가 작성 화면** - 동료 평가 입력 UI
2. **프로필 결과 화면** - 내 평가 결과 요약 대시보드

### C. Canva PPT (팀 발표용)
- 서비스 개요 + 경쟁사 분석 + 차별점 + MVP 계획

## 경쟁사 분석

### 국내

| 서비스 | 핵심 기능 | 우리와의 차이 |
|--------|----------|--------------|
| **스펙터** | 채용 시 레퍼런스 체크 (평판조회) | 지원자가 추천인 직접 지정 → 편향, 1회성 |
| **레몬베이스** | 내부 성과관리 (OKR, 피드백, 리뷰) | 내부 전용, 이직 시 활용 불가 |
| **클랩** | 중견기업 성과관리 솔루션 | 내부 전용, 외부 공유 불가 |
| **플렉스** | 올인원 HR 플랫폼 (근태, 급여 등) | 평가는 부가 기능, 핵심은 HRM |

### 해외

| 서비스 | 핵심 기능 | 우리와의 차이 |
|--------|----------|--------------|
| **Crosschq** | 360도 디지털 레퍼런스 체크 | 채용 시점 1회성, AI 분석 |
| **HiPeople** | AI 기반 레퍼런스 체크 자동화 | 채용 시점 1회성 |
| **Harver (Checkster)** | 자동화 레퍼런스 체크 | 채용 시점 1회성 |

### 경쟁 우위 요약
> **"재직 중 팀 전체 익명 평가를 축적하여 이직 시 활용"하는 서비스는 국내외 모두 없음**

### 리스크 & 대응

| 리스크 | 대응 |
|--------|------|
| 스펙터가 상시 평가 기능 추가 | "내부 피드백 문화" 포지셔닝 선점 |
| 레몬베이스/클랩이 외부 공유 기능 추가 | "이직 시 활용"이라는 독자적 가치 강조 |
| 동료평가 부정적 인식 (카카오 사례) | 부정적 문항 배제, 성장 중심 피드백 설계 |

## MVP 핵심 기능

```
1. 회사 등록 (관리자)
2. 팀원 등록/초대
3. 동료 평가 작성 (팀 전체, 익명)
4. 본인 평가 결과 확인
5. 외부 공유용 프로필 생성
```

### 평가 열람 권한
- 지원자가 "이 회사에 공개" 직접 허용하는 동의형 구조
- 전체 공개 or 전체 비공개 (항목별 선택 불가 → 신뢰도 확보)
- 평가 프로필이 없는 것 자체가 시그널로 작용
